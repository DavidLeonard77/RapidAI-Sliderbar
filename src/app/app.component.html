ANGULAR FRAMEWORK
For this I chose Angular and decided on creating a "dumb" component pattern to implement the feature. The reasoning is the component is small, re-usable and presentational.

OPTIONS
The was no requirement here but it seemed, at a minimum, this component should have the following set of configurable options and event emitters:

- Color(accessable through css)
- [value](optional): number
- [ticks](optional): number
- [snap](optional): boolean
- [name](optional): string

- (dragStarted): void
- (dragEnded): void
- (valueChange): number

Note: Also considered but not implemented:
- [disabled](optional:boolean
- [min](optional):number
- [max](optional):number

STRUCTURE
Chose svg (mostly) to render the graphics over pure css for scalability and options. Also, a resposive slider width (css) seemed prefferd over fixed.

NOTES
Angular change-detection stategy could be looked at further for performance gains. Also, unit test were not written for the sake of time but would normally be included for a feature that is considered "complete".
